## API Endpoint Details

### 1. Login Endpoint
**Path**: `/api/method/franchise_crm.api.auth_api.login`

**Request**:
```json
{
  "email": "user@example.com",
  "password": "secure_password"
}
```

**Success Response (200)**:
```json
{
  "message": {
    "status": "success",
    "message": "Login successful",
    "session_id": "abc123xyz789",
    "user": {
      "email": "user@example.com",
      "full_name": "User Name",
      "user_type": "Website User"
    }
  }
}
```

**Error Response (500)**:
```json
{
  "exc": "[\"frappe.exceptions.ValidationError\"]",
  "message": "Invalid email or password"
}
```

### 2. Signup Endpoint
**Path**: `/api/method/franchise_crm.api.auth_api.signup`

**Request**:
```json
{
  "email": "newuser@example.com",
  "password": "secure_password",
  "full_name": "New User"
}
```

**Success Response (200)**:
```json
{
  "message": {
    "status": "success",
    "message": "Signup successful",
    "session_id": "abc123xyz789",
    "user": {
      "email": "newuser@example.com",
      "full_name": "New User",
      "user_type": "Website User"
    }
  }
}
```

**Validation Errors**:
- "Email and password are required"
- "Password must be at least 8 characters long"
- "User with this email already exists"
- "Email is reserved"

### 3. Logout Endpoint
**Path**: `/api/method/franchise_crm.api.auth_api.logout`

**Request**: No parameters needed

**Success Response (200)**:
```json
{
  "message": {
    "status": "success",
    "message": "Logged out successfully"
  }
}
```

### 4. Get Current User Endpoint
**Path**: `/api/method/franchise_crm.api.auth_api.get_current_user`

**Request**: No parameters needed

**Response (Authenticated)**:
```json
{
  "message": {
    "status": "success",
    "user": {
      "email": "user@example.com",
      "full_name": "User Name",
      "user_type": "Website User",
      "enabled": true
    },
    "is_authenticated": true
  }
}
```

**Response (Not Authenticated)**:
```json
{
  "message": {
    "status": "success",
    "user": null,
    "is_authenticated": false
  }
}
```

## Database Operations

### User Creation Flow

When a user signs up:
1. Email is checked for uniqueness
2. New User document is created with:
   - email (primary key)
   - first_name (from full_name)
   - full_name
   - new_password (automatically hashed)
   - user_type = "Website User"
   - enabled = 1
3. User is automatically saved to database
4. Session is created
5. User is logged in

### Session Management

- Sessions are stored in `tabSessions` table
- Session ID (sid) is generated by Frappe
- Session status: "Active" or "Inactive"
- Used for authenticating subsequent requests

## Customization Options

### Modify Password Requirements

**File**: `franchise_crm/api/auth_api.py`

In `signup()` function, change:
```python
if len(password) < 8:
    frappe.throw(_("Password must be at least 8 characters long"), frappe.ValidationError)
```

To:
```python
if len(password) < 12:
    frappe.throw(_("Password must be at least 12 characters long"), frappe.ValidationError)
```

### Add Email Verification

Add this to `signup()` function:
```python
# Send verification email
frappe.sendmail(
    recipients=[email],
    subject="Verify your email",
    message=f"Click here to verify: {verification_link}"
)
```

### Add User Roles/Permissions

Modify `signup()` to add default roles:
```python
user.add_roles("role_name")
```

### Add User Metadata

Store additional info in User doctype:
```python
user.custom_field = "value"
```

```python
    session_id = get_session_id_from_headers()
    if not session_id:
        return
    
    # Try cache first
    cached_user = cache.get(f"session:{session_id}")
    if cached_user:
        frappe.set_user(cached_user)
        return
    
    # Fall back to database
    # ... rest of the code
```

## Production Deployment Considerations

1. **HTTPS Only**: Ensure all API calls use HTTPS in production
2. **Rate Limiting**: Add rate limiting to prevent brute force attacks
3. **CORS**: Restrict CORS to your specific frontend domain
4. **Session Timeout**: Configure appropriate session expiry
5. **Password Hashing**: Frappe handles this automatically
6. **CSRF Protection**: Already implemented by Frappe
7. **Monitoring**: Log all authentication attempts
8. **Backup**: Regular database backups for User data

## Additional Notes

- All passwords are automatically hashed using Frappe's built-in security
- Session IDs are generated by Frappe's session manager
- CSRF tokens are automatically validated on POST requests
- All API responses follow Frappe's standard format
- Error messages are sanitized to prevent information leakage

For more information, refer to the [Frappe Documentation](https://frappeframework.com/docs/user/en/basics)
